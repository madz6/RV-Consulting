# Repository instructions

This repository defines Pricing Strategy Workbench.

Before product, design or implementation work:

1. Read [`docs/product/README.md`](docs/product/README.md).
2. Use [`docs/product/PRD.md`](docs/product/PRD.md) as the source of truth for scope and acceptance criteria.
3. Preserve the product spine: decision framing → hypothesis/context decomposition → active research → reviewed evidence and analytical findings → human strategy co-creation → reproducible modelling → approved decision assets.
4. Keep retrieved facts, client assertions, human assumptions, AI inferences, calculated results and recommendations as distinct types.
5. Do not introduce cross-client retrieval, autonomous live-price execution, automated legal conclusions or unreviewed client-facing output.
6. Treat the hackathon demonstrator and pilotable MVP as different assurance levels.
7. Add or change P0 scope only by updating the PRD, relevant supporting document and validation plan together.

Use Markdown links between documents instead of duplicating canonical requirements.
